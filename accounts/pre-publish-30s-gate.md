# 知乎发布前 30 秒闸门

适用：L4 / L6 所有公开发布前；若账号已收到“可能包含 AI 创作内容”警告，则默认叠加执行恢复期专用闸门。
目标：把分散规则压成临门一脚检查卡，降低错发、AI 限流和账号错配风险。

## 一、所有账号通用（6 选 6，缺一不发）
1. 这篇内容是不是明显适合这个账号，而不是适合另一个号？
2. 这篇有没有明确主判断，而不是“有利有弊、还需观察”？
3. 这篇有没有具体场景或人物，而不是泛泛表达？
4. 这篇有没有可验证细节（截图 / 价格 / 实测 / 对比 / 历史经验）至少一种？
5. 这篇有没有明显 AI 套话、模板结构、过度工整节奏？
6. 这篇发出去，目标到底是流量、转化、自荐，还是投稿推进？如果目标说不清，不发。

## 二、L4 专用闸门
1. 第一段是否已经直接给判断，且优先由人工直写？
2. 是否写清谁适合、谁不适合？
3. 是否有真实取舍、坑点或失败经验？
4. 商品/工具是否自然出现，而不是像广告块？
5. 这篇是否会被看成“AI 工具综述模板文”？如果会，不发。

## 三、L6 专用闸门
1. 开头 100 字内是否出现人物 + 异常 + 冲突细节？
2. 是否有具体场景和物件，而不是抽象气氛？
3. 人物说话是否像真人，而不是工具人对白？
4. 文中是否至少有一次明显升级或反转？
5. 这篇是否像真人口述/故事，而不是工整 AI 短篇？如果像后者，不发。

## 四、警告账号补充口径（新增）
1. 若账号已被警告或近期连续折叠：宁可少发，不可错发。
2. 恢复期内容不得继续沿用被折叠内容的同款开头、同款结构、同款节奏。
3. 首段、结论段、关键取舍段不得完全交给 AI 生成。
4. 同一热点 48 小时内最多 1 篇主回答。
5. 若遮住账号名，这篇看起来仍像“上一批被折叠内容的同一生产线”，则不发。
6. 恢复期公开发布前，必须再通过 `pre-publish-recovery-gate.md`。

## 五、最终口径
- 通过闸门 ≠ 一定爆，但不通过闸门就不允许发布。
- 文稿完成不等于完成；公开发布 / 自荐 / 投稿推进才算真正动作。
- 警告账号当前优先级不是提量，而是停止新增风险样本、恢复真人信号和分发。

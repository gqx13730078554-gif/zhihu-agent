# 知乎发布前恢复期闸门（警告账号专用）

更新时间：2026-05-01 12:24 CST
适用：已收到“可能包含 AI 创作内容”警告的知乎账号。
目标：在恢复期把错误发布率压到最低，宁可少发，不可错发。

## 一、总闸门（8 选 8，缺一不发）
1. 这篇首段是不是人工直写，而不是 AI 一把生成？
2. 这篇有没有一句明确主判断，而不是“都对、都行、都看情况”？
3. 这篇有没有具体场景/人物/预算/设备/限制条件？
4. 这篇有没有至少 1 个可验证细节（截图 / 价格 / 参数 / 实测 / 失败经验 / 历史更新）？
5. 这篇有没有明确写出“谁适合、谁不适合”？
6. 这篇有没有真实取舍、坑点或反直觉判断？
7. 这篇是不是避开了明显 AI 套话、模板结构和过度工整节奏？
8. 这篇发出去的目标是不是单一且明确：流量 / 分发恢复 / 转化 / 自荐 / 投稿推进？说不清，不发。

## 二、恢复期直接否决条件（命中 1 条就不发）
- 首段像行业周报、百科摘要或模板化快评。
- 通篇没有证据，只有观点和总结。
- 先推商品/工具，再倒补理由。
- 看起来像“AI 工具综述模板文”或“热点解释模板文”。
- 同一热点 48 小时内已发过同类内容。
- 这篇如果遮住账号名，看起来和之前被折叠内容像同一生产线。
- 写完后仍说不清：这篇到底帮谁解决什么具体问题。

## 三、L4 额外闸门
1. 是否有真实使用场景，而不是泛泛推荐？
2. 是否至少写出 1 个不推荐理由？
3. 是否避免多商品、多工具同时堆砌？
4. 是否避免商业意图过于靠前、过于密集？

## 四、L6 额外闸门
1. 开头 100 字内是否出现异常细节、人物和冲突？
2. 人物说话是否像真人，而不是功能性对白？
3. 是否至少存在一次信息升级或反转？
4. 是否避免工整 AI 故事腔？

## 五、执行口径
- 通过闸门 ≠ 一定爆；不通过闸门 = 绝对不发。
- 恢复期优先“少而真”，不是“多而像”。
- 任何想发但闸门没过的内容，退回到“补证据 / 改首段 / 加取舍 / 改目标”流程。
